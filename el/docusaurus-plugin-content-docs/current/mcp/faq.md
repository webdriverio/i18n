Συχνά υποβαλλόμενες ερωτήσεις σχετικά με το WebdriverIO MCP.

## Γενικά

### Τι είναι το MCP;

Το MCP (Model Context Protocol) είναι ένα ανοιχτό πρωτόκολλο που επιτρέπει σε βοηθούς τεχνητής νοημοσύνης όπως το Claude να αλληλεπιδρούν με εξωτερικά εργαλεία και υπηρεσίες. Το WebdriverIO MCP υλοποιεί αυτό το πρωτόκολλο για να παρέχει δυνατότητες αυτοματισμού περιήγησης και κινητών συσκευών στο Claude Desktop και Claude Code.

### Τι μπορώ να αυτοματοποιήσω με το WebdriverIO MCP;

Μπορείτε να αυτοματοποιήσετε:
-   **Περιηγητές επιτραπέζιων υπολογιστών** (Chrome) - πλοήγηση, κλικ, πληκτρολόγηση, στιγμιότυπα οθόνης
-   **Εφαρμογές iOS** - σε προσομοιωτές ή πραγματικές συσκευές
-   **Εφαρμογές Android** - σε εξομοιωτές ή πραγματικές συσκευές
-   **Υβριδικές εφαρμογές** - εναλλαγή μεταξύ εγγενών και διαδικτυακών περιβαλλόντων

### Χρειάζεται να γράψω κώδικα;

Όχι! Αυτό είναι το κύριο πλεονέκτημα του MCP. Μπορείτε να περιγράψετε τι θέλετε να κάνετε σε φυσική γλώσσα, και το Claude θα χρησιμοποιήσει τα κατάλληλα εργαλεία για να εκτελέσει την εργασία.

**Παραδείγματα προτροπών:**
-   "Άνοιξε το Chrome και πλοηγήσου στο webdriver.io"
-   "Κάνε κλικ στο κουμπί Ξεκίνα"
-   "Τράβα ένα στιγμιότυπο της τρέχουσας σελίδας"
-   "Ξεκίνα την εφαρμογή iOS μου και συνδέσου ως δοκιμαστικός χρήστης"

## Εγκατάσταση & Ρύθμιση

### Πώς εγκαθιστώ το WebdriverIO MCP;

Δεν χρειάζεται να το εγκαταστήσετε ξεχωριστά. Ο διακομιστής MCP εκτελείται αυτόματα μέσω npx όταν το διαμορφώνετε στο Claude Desktop ή Claude Code.

Προσθέστε αυτό στη διαμόρφωση του Claude Desktop:

```json
{
    "mcpServers": {
        "wdio-mcp": {
            "command": "npx",
            "args": ["-y", "@wdio/mcp"]
        }
    }
}
```

### Πού βρίσκεται το αρχείο διαμόρφωσης του Claude Desktop;

-   **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
-   **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

### Χρειάζομαι το Appium για αυτοματισμό περιηγητή;

Όχι. Ο αυτοματισμός περιηγητή απαιτεί μόνο την εγκατάσταση του Chrome. Το WebdriverIO χειρίζεται αυτόματα το ChromeDriver.

### Χρειάζομαι το Appium για αυτοματισμό κινητών συσκευών;

Ναι. Ο αυτοματισμός κινητών συσκευών απαιτεί:
1. Διακομιστή Appium σε λειτουργία (`npm install -g appium && appium`)
2. Εγκατεστημένους οδηγούς πλατφορμών (`appium driver install xcuitest` για iOS, `appium driver install uiautomator2` για Android)
3. Κατάλληλα εργαλεία ανάπτυξης (Xcode για iOS, Android SDK για Android)

## Αυτοματισμός Περιηγητή

### Ποιοι περιηγητές υποστηρίζονται;

Επί του παρόντος, υποστηρίζεται μόνο ο **Chrome**. Η υποστήριξη για άλλους περιηγητές μπορεί να προστεθεί σε μελλοντικές εκδόσεις.

### Μπορώ να εκτελέσω το Chrome σε λειτουργία χωρίς κεφαλή (headless);

Ναι! Ζητήστε από το Claude να ξεκινήσει τον περιηγητή σε λειτουργία χωρίς κεφαλή:

"Ξεκίνα το Chrome σε λειτουργία χωρίς κεφαλή"

Ή το Claude θα χρησιμοποιήσει αυτήν την επιλογή όταν είναι κατάλληλη (π.χ., σε πλαίσια CI/CD).

### Μπορώ να ορίσω το μέγεθος του παραθύρου του περιηγητή;

Ναι. Μπορείτε να καθορίσετε διαστάσεις κατά την εκκίνηση του περιηγητή:

"Ξεκίνα το Chrome με μέγεθος παραθύρου 1920x1080"

Υποστηριζόμενες διαστάσεις: 400-3840 pixel πλάτος, 400-2160 pixel ύψος. Η προεπιλογή είναι 1920x1080.

### Μπορώ να ξεκινήσω τον περιηγητή και να πλοηγηθώ σε ένα βήμα;

Ναι! Χρησιμοποιήστε την παράμετρο `navigationUrl`:

"Ξεκίνα το Chrome και πλοηγήσου στο https://webdriver.io"

Αυτό είναι πιο αποτελεσματικό από το να ξεκινήσετε τον περιηγητή και μετά να πλοηγηθείτε ξεχωριστά.

### Πώς τραβάω στιγμιότυπα οθόνης;

Απλά ζητήστε από το Claude:

"Τράβα ένα στιγμιότυπο της τρέχουσας σελίδας"

Τα στιγμιότυπα οθόνης βελτιστοποιούνται αυτόματα:
- Κλιμακώνονται σε μέγιστη διάσταση 2000px
- Συμπιέζονται σε μέγιστο μέγεθος αρχείου 1MB
- Μορφή: PNG ή JPEG (επιλέγεται αυτόματα για βέλτιστη ποιότητα)

### Μπορώ να αλληλεπιδράσω με iframes;

Επί του παρόντος, ο διακομιστής MCP λειτουργεί στο κύριο έγγραφο. Η αλληλεπίδραση με iframe μπορεί να προστεθεί σε μελλοντικές εκδόσεις.

### Μπορώ να εκτελέσω προσαρμοσμένη JavaScript;

Ναι! Χρησιμοποιήστε το εργαλείο `execute_script`:

"Εκτέλεσε script για να πάρεις τον τίτλο της σελίδας"
"Εκτέλεσε script: return document.querySelectorAll('button').length"

## Αυτοματισμός Κινητών

### Πώς ξεκινάω μια εφαρμογή iOS;

Ζητήστε από το Claude με τις απαραίτητες λεπτομέρειες:

"Ξεκίνα την εφαρμογή iOS μου που βρίσκεται στο /path/to/MyApp.app στον προσομοιωτή iPhone 15"

Ή για μια εγκατεστημένη εφαρμογή:

"Ξεκίνα την εφαρμογή με ενεργοποιημένο το noReset στον προσομοιωτή iPhone 15"

### Πώς ξεκινάω μια εφαρμογή Android;

"Ξεκίνα την εφαρμογή Android μου στο /path/to/app.apk στον εξομοιωτή Pixel 7"

Ή για μια εγκατεστημένη εφαρμογή:

"Ξεκίνα την εφαρμογή με ενεργοποιημένο το noReset στον εξομοιωτή Pixel 7"

### Μπορώ να κάνω δοκιμές σε πραγματικές συσκευές;

Ναι! Για πραγματικές συσκευές, θα χρειαστείτε το UDID της συσκευής:

-   **iOS:** Συνδέστε τη συσκευή, ανοίξτε το Finder, κάντε κλικ στη συσκευή, κάντε κλικ στον αριθμό σειράς για να αποκαλυφθεί το UDID
-   **Android:** Εκτελέστε `adb devices` στο τερματικό

Στη συνέχεια ζητήστε από το Claude:

"Ξεκίνα την εφαρμογή iOS μου στην πραγματική συσκευή με UDID abc123..."

### Πώς χειρίζομαι διαλόγους αδειών;

Από προεπιλογή, οι άδειες χορηγούνται αυτόματα (`autoGrantPermissions: true`). Αν χρειάζεται να δοκιμάσετε ροές αδειών, μπορείτε να το απενεργοποιήσετε:

"Ξεκίνα την εφαρμογή μου χωρίς αυτόματη χορήγηση αδειών"

### Ποιες χειρονομίες υποστηρίζονται;

-   **Πάτημα:** Πάτημα σε στοιχεία ή συντεταγμένες
-   **Σάρωση:** Σάρωση προς τα πάνω, κάτω, αριστερά ή δεξιά
-   **Σύρε και Άφησε:** Σύρετε από ένα στοιχείο σε άλλο ή σε συντεταγμένες

Σημείωση: Το `long_press` είναι διαθέσιμο μέσω του `execute_script` με εντολές Appium για κινητά.

### Πώς κάνω κύλιση σε εφαρμογές κινητών;

Χρησιμοποιήστε χειρονομίες σάρωσης:

"Σάρωσε προς τα πάνω για κύλιση προς τα κάτω"
"Σάρωσε προς τα κάτω για κύλιση προς τα πάνω"

### Μπορώ να περιστρέψω τη συσκευή;

Ναι:

"Περιέστρεψε τη συσκευή σε οριζόντια προβολή"
"Περιέστρεψε τη συσκευή σε κατακόρυφη προβολή"

### Πώς χειρίζομαι υβριδικές εφαρμογές;

Για εφαρμογές με webviews, μπορείτε να αλλάξετε πλαίσια:

"Δείξε τα διαθέσιμα πλαίσια"
"Άλλαξε στο πλαίσιο webview"
"Επέστρεψε στο εγγενές πλαίσιο"

### Μπορώ να εκτελέσω εντολές Appium για κινητά;

Ναι! Χρησιμοποιήστε το εργαλείο `execute_script`:

```
Execute script "mobile: pressKey" with args [{ keycode: 4 }]  // Πάτημα BACK στο Android
Execute script "mobile: activateApp" with args [{ appId: "com.example.app" }]
Execute script "mobile: terminateApp" with args [{ bundleId: "com.example.app" }]
```

## Επιλογή Στοιχείων

### Πώς γνωρίζει το Claude ποιο στοιχείο να χρησιμοποιήσει;

Το Claude χρησιμοποιεί το εργαλείο `get_visible_elements` για να εντοπίσει διαδραστικά στοιχεία στη σελίδα/οθόνη. Κάθε στοιχείο συνοδεύεται από πολλαπλές στρατηγικές επιλογής.

### Τι γίνεται αν υπάρχουν πάρα πολλά στοιχεία στη σελίδα;

Χρησιμοποιήστε σελιδοποίηση για τη διαχείριση μεγάλων λιστών στοιχείων:

"Πάρε τα πρώτα 20 ορατά στοιχεία"
"Πάρε ορατά στοιχεία με μετατόπιση 20 και όριο 20"

Η απάντηση περιλαμβάνει `total`, `showing` και `hasMore` για να βοηθήσει στην πλοήγηση μέσα από τα στοιχεία.

### Μπορώ να πάρω μόνο συγκεκριμένους τύπους στοιχείων;

Ναι! Χρησιμοποιήστε την παράμετρο `elementType`:

-   `interactable` (προεπιλογή): Κουμπιά, σύνδεσμοι, πεδία εισαγωγής
-   `visual`: Εικόνες, SVGs
-   `all`: Και οι δύο τύποι

"Πάρε ορατά οπτικά στοιχεία στη σελίδα"

### Τι γίνεται αν το Claude κάνει κλικ στο λάθος στοιχείο;

Μπορείτε να είστε πιο συγκεκριμένοι:

-   Δώστε ακριβές κείμενο: "Κάνε κλικ στο κουμπί που γράφει 'Υποβολή Παραγγελίας'"
-   Δώστε επιλογέα: "Κάνε κλικ στο στοιχείο με επιλογέα #submit-btn"
-   Δώστε αναγνωριστικό προσβασιμότητας: "Κάνε κλικ στο στοιχείο με αναγνωριστικό προσβασιμότητας loginButton"

### Ποια είναι η καλύτερη στρατηγική επιλογής για κινητά;

1. **Αναγνωριστικό Προσβασιμότητας** (καλύτερο) - `~loginButton`
2. **Αναγνωριστικό Πόρου** (Android) - `id=login_button`
3. **Συμβολοσειρά Κατηγορήματος** (iOS) - `-ios predicate string:label == "Login"`
4. **XPath** (τελευταία λύση) - πιο αργό αλλά λειτουργεί παντού

### Τι είναι το δέντρο προσβασιμότητας και πότε πρέπει να το χρησιμοποιώ;

Το δέντρο προσβασιμότητας παρέχει σημασιολογικές πληροφορίες για τα στοιχεία της σελίδας (ρόλοι, ονόματα, καταστάσεις). Χρησιμοποιήστε το `get_accessibility` όταν:
- Το `get_visible_elements` δεν επιστρέφει τα αναμενόμενα στοιχεία
- Χρειάζεται να βρείτε στοιχεία με ρόλο προσβασιμότητας (κουμπί, σύνδεσμος, πλαίσιο κειμένου, κλπ.)
- Χρειάζεστε λεπτομερείς σημασιολογικές πληροφορίες για τα στοιχεία

"Πάρε το δέντρο προσβασιμότητας φιλτραρισμένο για ρόλους κουμπιού και συνδέσμου"

## Διαχείριση Συνεδρίας

### Μπορώ να έχω πολλές συνεδρίες ταυτόχρονα;

Όχι. Ο διακομιστής MCP χρησιμοποιεί μοντέλο μονής συνεδρίας. Μόνο μία συνεδρία περιηγητή ή εφαρμογής μπορεί να είναι ενεργή κάθε φορά.

### Τι συμβαίνει όταν κλείνω μια συνεδρία;

Εξαρτάται από τον τύπο της συνεδρίας και τις ρυθμίσεις:

-   **Περιηγητής:** Το Chrome κλείνει εντελώς
-   **Κινητό με `noReset: false`:** Η εφαρμογή τερματίζεται
-   **Κινητό με `noReset: true` ή χωρίς `appPath`:** Η εφαρμογή παραμένει ανοιχτή (η συνεδρία αποσυνδέεται αυτόματα)

### Μπορώ να διατηρήσω την κατάσταση της εφαρμογής μεταξύ συνεδριών;

Ναι! Χρησιμοποιήστε την επιλογή `noReset`:

"Ξεκίνα την εφαρμογή μου με ενεργοποιημένο το noReset"

Αυτό διατηρεί την κατάσταση σύνδεσης, τις προτιμήσεις και άλλα δεδομένα εφαρμογής.

### Ποια είναι η διαφορά μεταξύ κλεισίματος και αποσύνδεσης;

-   **Κλείσιμο:** Τερματίζει εντελώς τον περιηγητή/την εφαρμογή
-   **Αποσύνδεση:** Αποσυνδέει τον αυτοματισμό αλλά διατηρεί τον περιηγητή/την εφαρμογή σε λειτουργία

Η αποσύνδεση είναι χρήσιμη όταν θέλετε να επιθεωρήσετε χειροκίνητα την κατάσταση μετά τον αυτοματισμό.

### Η συνεδρία μου εξακολουθεί να λήγει κατά τη διάρκεια της αποσφαλμάτωσης

Αυξήστε το χρονικό όριο εντολής:

"Ξεκίνα την εφαρμογή μου με newCommandTimeout 300 δευτερολέπτων"

Η προεπιλογή είναι 60 δευτερόλεπτα. Για μεγάλες συνεδρίες αποσφαλμάτωσης, δοκιμάστε 300-600 δευτερόλεπτα.

## Αντιμετώπιση Προβλημάτων

### Σφάλμα "Session not found"

Αυτό σημαίνει ότι δεν υπάρχει ενεργή συνεδρία. Ξεκινήστε πρώτα μια συνεδρία περιηγητή ή εφαρμογής:

"Ξεκίνα το Chrome και πλοηγήσου στο google.com"

### Σφάλμα "Element not found"

Το στοιχείο μπορεί να μην είναι ορατό ή μπορεί να έχει διαφορετικό επιλογέα. Δοκιμάστε:

1. Να ζητήσετε από το Claude να πάρει πρώτα όλα τα ορατά στοιχεία
2. Να παρέχετε πιο συγκεκριμένο επιλογέα
3. Να περιμένετε να φορτώσει πλήρως η σελίδα/εφαρμογή
4. Να χρησιμοποιήσετε `inViewportOnly: false` για να βρείτε στοιχεία εκτός οθόνης

### Ο περιηγητής δεν ξεκινά

1. Βεβαιωθείτε ότι το Chrome είναι εγκατεστημένο
2. Ελέγξτε αν μια άλλη διαδικασία χρησιμοποιεί τη θύρα αποσφαλμάτωσης (9222)
3. Δοκιμάστε τη λειτουργία χωρίς κεφαλή

### Η σύνδεση με το Appium απέτυχε

Αυτό είναι το πιο συνηθισμένο πρόβλημα κατά την έναρξη αυτοματισμού κινητών.

1. **Επαληθεύστε ότι το Appium εκτελείται**: `curl http://localhost:4723/status`
2. Ξεκινήστε το Appium εάν χρειάζεται: `appium`
3. Ελέγξτε ότι η διαμόρφωση του URL Appium ταιριάζει με το διακομιστή
4. Βεβαιωθείτε ότι οι οδηγοί είναι εγκατεστημένοι: `appium driver list --installed`

:::tip
Ο διακομιστής MCP απαιτεί το Appium να εκτελείται πριν από την έναρξη συνεδριών κινητών. Φροντίστε να ξεκινήσετε πρώτα το Appium:
```sh
appium
```
Οι μελλοντικές εκδόσεις ενδέχεται να περιλαμβάνουν αυτόματη διαχείριση υπηρεσίας Appium.
:::

### Ο προσομοιωτής iOS δεν ξεκινά

1. Βεβαιωθείτε ότι το Xcode είναι εγκατεστημένο: `xcode-select --install`
2. Δείτε τους διαθέσιμους προσομοιωτές: `xcrun simctl list devices`
3. Ελέγξτε για συγκεκριμένα σφάλματα προσομοιωτή στο Console.app

### Ο εξομοιωτής Android δεν ξεκινά

1. Ορίστε το `ANDROID_HOME`: `export ANDROID_HOME=$HOME/Library/Android/sdk`
2. Ελέγξτε τους εξομοιωτές: `emulator -list-avds`
3. Ξεκινήστε χειροκίνητα τον εξομοιωτή: `emulator -avd <avd-name>`
4. Επαληθεύστε ότι η συσκευή είναι συνδεδεμένη: `adb devices`

### Τα στιγμιότυπα οθόνης δεν λειτουργούν

1. Για κινητά, βεβαιωθείτε ότι η συνεδρία είναι ενεργή
2. Για περιηγητή, δοκιμάστε μια διαφορετική σελίδα (ορισμένες σελίδες εμποδίζουν τα στιγμιότυπα οθόνης)
3. Ελέγξτε τα αρχεία καταγραφής του Claude Desktop για σφάλματα

Τα στιγμιότυπα οθόνης συμπιέζονται αυτόματα σε μέγιστο 1MB, επομένως τα μεγάλα στιγμιότυπα θα λειτουργούν αλλά ενδέχεται να έχουν χαμηλότερη ποιότητα.

## Απόδοση

### Γιατί ο αυτοματισμός κινητών είναι αργός;

Ο αυτοματισμός κινητών περιλαμβάνει:
1. Επικοινωνία δικτύου με το διακομιστή Appium
2. Το Appium που επικοινωνεί με τη συσκευή/προσομοιωτή
3. Απόδοση και απόκριση της συσκευής

Συμβουλές για ταχύτερο αυτοματισμό:
-   Χρησιμοποιήστε εξομοιωτές/προσομοιωτές αντί για πραγματικές συσκευές για ανάπτυξη
-   Χρησιμοποιήστε αναγνωριστικά προσβασιμότητας αντί για XPath
-   Ενεργοποιήστε το `inViewportOnly: true` για εντοπισμό στοιχείων
-   Χρησιμοποιήστε σελιδοποίηση (`limit`) για να μειώσετε τη χρήση token

### Πώς μπορώ να επιταχύνω τον εντοπισμό στοιχείων;

Ο διακομιστής MCP ήδη βελτιστοποιεί τον εντοπισμό στοιχείων χρησιμοποιώντας ανάλυση XML πηγής σελίδας (2 HTTP κλήσεις έναντι 600+ για παραδοσιακά ερωτήματα στοιχείων). Πρόσθετες συμβουλές:

-   Διατηρήστε το `inViewportOnly: true` (προεπιλογή)
-   Ορίστε `includeContainers: false` (προεπιλογή)
-   Χρησιμοποιήστε `limit` και `offset` για σελιδοποίηση σε μεγάλες οθόνες
-   Χρησιμοποιήστε συγκεκριμένους επιλογείς αντί να βρίσκετε όλα τα στοιχεία

### Τα στιγμιότυπα οθόνης είναι αργά ή αποτυγχάνουν

Τα στιγμιότυπα οθόνης βελτιστοποιούνται αυτόματα:
- Αλλάζουν μέγεθος εάν είναι μεγαλύτερα από 2000px
- Συμπιέζονται για να παραμείνουν κάτω από 1MB
- Μετατρέπονται σε JPEG εάν το PNG είναι πολύ μεγάλο

Αυτή η βελτιστοποίηση μειώνει το χρόνο επεξεργασίας και διασφαλίζει ότι το Claude μπορεί να χειριστεί την εικόνα.

## Περιορισμοί

### Ποιοι είναι οι τρέχοντες περιορισμοί;

-   **Μονή συνεδρία:** Μόνο ένας περιηγητής/εφαρμογή κάθε φορά
-   **Υποστήριξη περιηγητή:** Μόνο Chrome (προς το παρόν)
-   **Υποστήριξη iframe:** Περιορισμένη υποστήριξη για iframes
-   **Μεταφορτώσεις αρχείων:** Δεν υποστηρίζονται άμεσα μέσω εργαλείων
-   **Ήχος/Βίντεο:** Δεν μπορεί να αλληλεπιδράσει με αναπαραγωγή πολυμέσων
-   **Επεκτάσεις περιηγητή:** Δεν υποστηρίζονται

### Μπορώ να το χρησιμοποιήσω για δοκιμές παραγωγής;

Το WebdriverIO MCP είναι σχεδιασμένο για διαδραστικό αυτοματισμό με τη βοήθεια τεχνητής νοημοσύνης. Για δοκιμές παραγωγής CI/CD, εξετάστε τη χρήση του παραδοσιακού δρομολογητή δοκιμών του WebdriverIO με πλήρη προγραμματιστικό έλεγχο.

## Ασφάλεια

### Είναι ασφαλή τα δεδομένα μου;

Ο διακομιστής MCP εκτελείται τοπικά στον υπολογιστή σας. Όλος ο αυτοματισμός γίνεται μέσω τοπικών συνδέσεων περιηγητή/Appium. Δεν αποστέλλονται δεδομένα σε εξωτερικούς διακομιστές πέρα από αυτά στα οποία πλοηγείστε ρητά.

### Μπορεί το Claude να έχει πρόσβαση στους κωδικούς πρόσβασής μου;

Το Claude μπορεί να βλέπει το περιεχόμενο της σελίδας και να αλληλεπιδρά με στοιχεία, αλλά:
-   Οι κωδικοί πρόσβασης σε πεδία `<input type="password">` είναι καλυμμένοι
-   Θα πρέπει να αποφεύγετε τον αυτοματισμό ευαίσθητων διαπιστευτηρίων
-   Χρησιμοποιήστε δοκιμαστικούς λογαριασμούς για αυτοματισμό
