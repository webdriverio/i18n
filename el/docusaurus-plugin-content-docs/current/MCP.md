## Τι μπορεί να κάνει;

Το WebdriverIO MCP είναι ένας **διακομιστής Model Context Protocol (MCP)** που επιτρέπει σε βοηθούς τεχνητής νοημοσύνης όπως το Claude Desktop και το Claude Code να αυτοματοποιούν και να αλληλεπιδρούν με προγράμματα περιήγησης ιστού και εφαρμογές για κινητά.

### Γιατί το WebdriverIO MCP;

-   **Mobile-First**: Σε αντίθεση με τους διακομιστές MCP μόνο για προγράμματα περιήγησης, το WebdriverIO MCP υποστηρίζει αυτοματισμό εγγενών εφαρμογών iOS και Android μέσω Appium
-   **Cross-Platform Selectors**: Η έξυπνη ανίχνευση στοιχείων δημιουργεί αυτόματα πολλαπλές στρατηγικές εντοπισμού (accessibility ID, XPath, UiAutomator, iOS predicates)
-   **Οικοσύστημα WebdriverIO**: Βασισμένο στο δοκιμασμένο πλαίσιο WebdriverIO με το πλούσιο οικοσύστημα υπηρεσιών και αναφορών του

Παρέχει ένα ενοποιημένο περιβάλλον για:

-   🖥️ **Προγράμματα περιήγησης υπολογιστή** (Chrome - με ή χωρίς περιβάλλον)
-   📱 **Εγγενείς εφαρμογές για κινητά** (Προσομοιωτές iOS / Εξομοιωτές Android / Πραγματικές συσκευές μέσω Appium)
-   📳 **Υβριδικές εφαρμογές για κινητά** (Εναλλαγή μεταξύ Native + WebView μέσω Appium)

μέσω του πακέτου [`@wdio/mcp`](https://www.npmjs.com/package/@wdio/mcp).

Αυτό επιτρέπει στους βοηθούς τεχνητής νοημοσύνης να:

-   **Εκκινούν και ελέγχουν προγράμματα περιήγησης** με ρυθμιζόμενες διαστάσεις, λειτουργία headless και προαιρετική αρχική πλοήγηση
-   **Περιηγούνται σε ιστοσελίδες** και να αλληλεπιδρούν με στοιχεία (κλικ, πληκτρολόγηση, κύλιση)
-   **Αναλύουν περιεχόμενο σελίδας** μέσω δέντρου προσβασιμότητας και ανίχνευσης ορατών στοιχείων με υποστήριξη σελιδοποίησης
-   **Λαμβάνουν στιγμιότυπα οθόνης** αυτόματα βελτιστοποιημένα (αλλαγή μεγέθους, συμπίεση μέχρι 1MB)
-   **Διαχειρίζονται cookies** για χειρισμό συνεδριών
-   **Ελέγχουν κινητές συσκευές** συμπεριλαμβανομένων χειρονομιών (άγγιγμα, σάρωση, σύρσιμο και απόθεση)
-   **Εναλλάσσουν περιβάλλοντα** σε υβριδικές εφαρμογές μεταξύ native και webview
-   **Εκτελούν σενάρια** - JavaScript σε προγράμματα περιήγησης, εντολές Appium για κινητές συσκευές
-   **Χειρίζονται λειτουργίες συσκευών** όπως περιστροφή, πληκτρολόγιο, γεωτοποθεσία
-   και πολλά άλλα, δείτε τις επιλογές [Εργαλείων](./mcp/tools) και [Διαμόρφωσης](./mcp/configuration)

:::info

ΣΗΜΕΙΩΣΗ Για Εφαρμογές Κινητών
Ο αυτοματισμός κινητών απαιτεί έναν εκτελούμενο διακομιστή Appium με τους κατάλληλους οδηγούς εγκατεστημένους. Δείτε τις [Προϋποθέσεις](#prerequisites) για οδηγίες εγκατάστασης.

## Εγκατάσταση

Ο ευκολότερος τρόπος να χρησιμοποιήσετε το `@wdio/mcp` είναι μέσω npx χωρίς τοπική εγκατάσταση:

```sh
npx @wdio/mcp
```

Ή εγκαταστήστε το καθολικά:

```sh
npm install -g @wdio/mcp
```

## Χρήση με το Claude

Για να χρησιμοποιήσετε το WebdriverIO MCP με το Claude, τροποποιήστε το αρχείο διαμόρφωσης:

```json
{
    "mcpServers": {
        "wdio-mcp": {
            "command": "npx",
            "args": ["-y", "@wdio/mcp"]
        }
    }
}
```

Μετά την προσθήκη της διαμόρφωσης, επανεκκινήστε το Claude. Τα εργαλεία WebdriverIO MCP θα είναι διαθέσιμα για εργασίες αυτοματισμού προγραμμάτων περιήγησης και κινητών.

### Χρήση με το Claude Code

Το Claude Code ανιχνεύει αυτόματα διακομιστές MCP. Μπορείτε να το διαμορφώσετε στο `.claude/settings.json` του έργου σας, ή στο `.mcp.json`.

Ή προσθέστε το στο .claude.json καθολικά εκτελώντας:
```bash
claude mcp add --transport stdio wdio-mcp -- npx -y @wdio/mcp
```
Επαληθεύστε το εκτελώντας την εντολή `/mcp` μέσα στο claude code.

## Προϋποθέσεις

### Αυτοματισμός Προγράμματος Περιήγησης

-   Το **Chrome** πρέπει να είναι εγκατεστημένο στο σύστημά σας
-   Το WebdriverIO διαχειρίζεται αυτόματα το ChromeDriver

### Αυτοματισμός Κινητών

#### iOS

1. **Εγκατάσταση Xcode** από το Mac App Store
2. **Εγκατάσταση Εργαλείων Γραμμής Εντολών Xcode**:
   ```sh
   xcode-select --install
   ```
3. **Εγκατάσταση Appium**:
   ```sh
   npm install -g appium
   ```
4. **Εγκατάσταση οδηγού XCUITest**:
   ```sh
   appium driver install xcuitest
   ```
5. **Εκκίνηση διακομιστή Appium**:
   ```sh
   appium
   ```
6. **Για Προσομοιωτές**: Άνοιγμα Xcode → Window → Devices and Simulators για δημιουργία/διαχείριση προσομοιωτών
7. **Για Πραγματικές Συσκευές**: Θα χρειαστείτε το UDID της συσκευής (40-χαρακτήρων μοναδικό αναγνωριστικό)

#### Android

1. **Εγκατάσταση Android Studio** και ρύθμιση Android SDK
2. **Ορισμός μεταβλητών περιβάλλοντος**:
   ```sh
   export ANDROID_HOME=$HOME/Library/Android/sdk
   export PATH=$PATH:$ANDROID_HOME/emulator
   export PATH=$PATH:$ANDROID_HOME/platform-tools
   ```
3. **Εγκατάσταση Appium**:
   ```sh
   npm install -g appium
   ```
4. **Εγκατάσταση οδηγού UiAutomator2**:
   ```sh
   appium driver install uiautomator2
   ```
5. **Εκκίνηση διακομιστή Appium**:
   ```sh
   appium
   ```
6. **Δημιουργία εξομοιωτή** μέσω Android Studio → Virtual Device Manager
7. **Εκκίνηση εξομοιωτή** πριν την εκτέλεση δοκιμών

## Αρχιτεκτονική

### Πώς Λειτουργεί

Το WebdriverIO MCP λειτουργεί ως γέφυρα μεταξύ βοηθών τεχνητής νοημοσύνης και αυτοματισμού προγραμμάτων περιήγησης/κινητών:

```
┌─────────────────┐     MCP Protocol      ┌─────────────────┐
│  Claude Desktop │ ◄──────────────────►  │    @wdio/mcp    │
│  or Claude Code │      (stdio)          │     Server      │
└─────────────────┘                       └────────┬────────┘
                                                   │
                                             WebDriverIO API
                                                   │
                    ┌──────────────────────────────┼──────────────────────────────┐
                    │                              │                              │
            ┌───────▼───────┐             ┌───────▼───────┐             ┌───────▼───────┐
            │    Chrome     │             │    Appium     │             │    Appium     │
            │   (Browser)   │             │     (iOS)     │             │   (Android)   │
            └───────────────┘             └───────────────┘             └───────────────┘
```

### Διαχείριση Συνεδρίας

-   **Μοντέλο μονής συνεδρίας**: Μόνο μία συνεδρία προγράμματος περιήγησης Ή εφαρμογής μπορεί να είναι ενεργή κάθε φορά
-   **Κατάσταση συνεδρίας** διατηρείται καθολικά σε όλες τις κλήσεις εργαλείων
-   **Αυτόματη αποσύνδεση**: Οι συνεδρίες με διατηρημένη κατάσταση (`noReset: true`) αποσυνδέονται αυτόματα κατά το κλείσιμο

### Ανίχνευση Στοιχείων

#### Πρόγραμμα Περιήγησης (Web)

-   Χρησιμοποιεί βελτιστοποιημένο script περιηγητή για εύρεση όλων των ορατών, αλληλεπιδραστικών στοιχείων
-   Επιστρέφει στοιχεία με επιλογείς CSS, IDs, classes και πληροφορίες ARIA
-   Φιλτράρει τα ορατά στο viewport στοιχεία από προεπιλογή

#### Κινητά (Εγγενείς Εφαρμογές)

-   Χρησιμοποιεί αποδοτική ανάλυση XML πηγής σελίδας (2 κλήσεις HTTP έναντι 600+ για παραδοσιακά ερωτήματα)
-   Ταξινόμηση στοιχείων ειδική για κάθε πλατφόρμα για Android και iOS
-   Δημιουργεί πολλαπλές στρατηγικές εντοπισμού ανά στοιχείο:
    -   Accessibility ID (διαπλατφορμικό, πιο σταθερό)
    -   Resource ID / Χαρακτηριστικό Name
    -   Αντιστοίχιση κειμένου / ετικέτας
    -   XPath (πλήρες και απλοποιημένο)
    -   UiAutomator (Android) / Predicates (iOS)

## Σύνταξη Επιλογέα

Ο διακομιστής MCP υποστηρίζει πολλαπλές στρατηγικές επιλογέων. Δείτε [Selectors](./mcp/selectors) για λεπτομερή τεκμηρίωση.

### Web (CSS/XPath)

```
# CSS Selectors
button.my-class
#element-id
[data-testid="login"]

# XPath
//button[@class='submit']
//a[contains(text(), 'Click')]

# Text Selectors (WebdriverIO specific)
button=Exact Button Text
a*=Partial Link Text
```

### Κινητά (Cross-Platform)

```
# Accessibility ID (recommended - works on iOS & Android)
~loginButton

# Android UiAutomator
android=new UiSelector().text("Login")

# iOS Predicate String
-ios predicate string:label == "Login"

# iOS Class Chain
-ios class chain:**/XCUIElementTypeButton[`label == "Login"`]

# XPath (works on both platforms)
//android.widget.Button[@text="Login"]
//XCUIElementTypeButton[@label="Login"]
```

## Αυτόματος Χειρισμός

### Δικαιώματα

Από προεπιλογή, ο διακομιστής MCP παραχωρεί αυτόματα δικαιώματα εφαρμογών (`autoGrantPermissions: true`), εξαλείφοντας την ανάγκη χειροκίνητου χειρισμού διαλόγων δικαιωμάτων κατά τον αυτοματισμό.

### Ειδοποιήσεις Συστήματος

Οι ειδοποιήσεις συστήματος (όπως "Να επιτραπούν οι ειδοποιήσεις;") γίνονται αυτόματα αποδεκτές από προεπιλογή (`autoAcceptAlerts: true`). Αυτό μπορεί να διαμορφωθεί ώστε να τις απορρίπτει με `autoDismissAlerts: true`.

## Διαμόρφωση

### Μεταβλητές Περιβάλλοντος

Διαμορφώστε τη σύνδεση διακομιστή Appium:

| Μεταβλητή | Προεπιλογή | Περιγραφή |
|----------|---------|-------------|
| `APPIUM_URL` | `127.0.0.1` | Όνομα κεντρικού υπολογιστή διακομιστή Appium |
| `APPIUM_URL_PORT` | `4723` | Θύρα διακομιστή Appium |
| `APPIUM_PATH` | `/` | Διαδρομή διακομιστή Appium |

## Βελτιστοποίηση Απόδοσης

Ο διακομιστής MCP είναι βελτιστοποιημένος για αποδοτική επικοινωνία με βοηθούς τεχνητής νοημοσύνης:

-   **Μορφή TOON**: Χρησιμοποιεί Token-Oriented Object Notation για ελάχιστη χρήση token
-   **Ανάλυση XML**: Η ανίχνευση στοιχείων κινητών χρησιμοποιεί 2 κλήσεις HTTP (έναντι 600+ παραδοσιακά)
-   **Συμπίεση Στιγμιοτύπων**: Εικόνες αυτόματα συμπιεσμένες έως 1MB χρησιμοποιώντας Sharp
-   **Φιλτράρισμα Viewport**: Επιστρέφονται μόνο ορατά στοιχεία από προεπιλογή
-   **Σελιδοποίηση**: Μεγάλες λίστες στοιχείων μπορούν να σελιδοποιηθούν για μείωση του μεγέθους απόκρισης

## Υποστήριξη TypeScript

Ο διακομιστής MCP είναι γραμμένος σε TypeScript και περιλαμβάνει πλήρεις ορισμούς τύπων. Εάν επεκτείνετε ή ενσωματώνετε τον διακομιστή προγραμματιστικά, θα επωφεληθείτε από την αυτόματη συμπλήρωση και την ασφάλεια τύπων.

### Το πρόγραμμα περιήγησης δεν ξεκινά

-   Βεβαιωθείτε ότι το Chrome είναι εγκατεστημένο
-   Ελέγξτε ότι καμία άλλη διαδικασία δεν χρησιμοποιεί την προεπιλεγμένη θύρα αποσφαλμάτωσης (9222)
-   Δοκιμάστε λειτουργία headless αν προκύπτουν προβλήματα οθόνης

### Αποτυχία σύνδεσης Appium

-   Επαληθεύστε ότι ο διακομιστής Appium εκτελείται (`appium`)
-   Ελέγξτε τη διαμόρφωση URL και θύρας Appium
-   Βεβαιωθείτε ότι ο κατάλληλος οδηγός είναι εγκατεστημένος (`appium driver list`)

### Προβλήματα με Προσομοιωτή iOS

-   Βεβαιωθείτε ότι το Xcode είναι εγκατεστημένο και ενημερωμένο
-   Ελέγξτε ότι οι προσομοιωτές είναι διαθέσιμοι (`xcrun simctl list devices`)
-   Για πραγματικές συσκευές, επαληθεύστε ότι το UDID είναι σωστό

### Προβλήματα με Εξομοιωτή Android

-   Βεβαιωθείτε ότι το Android SDK είναι σωστά διαμορφωμένο
-   Επαληθεύστε ότι ο εξομοιωτής εκτελείται (`adb devices`)
-   Ελέγξτε ότι η μεταβλητή περιβάλλοντος `ANDROID_HOME` έχει οριστεί
