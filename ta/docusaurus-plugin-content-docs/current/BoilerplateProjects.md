காலப்போக்கில், நமது சமூகம் பல திட்டங்களை உருவாக்கியுள்ளது, அவற்றை உங்கள் சொந்த சோதனைத் தொகுப்பை அமைக்க உத்வேகமாகப் பயன்படுத்தலாம்.

# v9 தயார்நிலை திட்டங்கள்

## [webdriverio/cucumber-boilerplate](https://github.com/webdriverio/cucumber-boilerplate)

Cucumber சோதனைத் தொகுப்புகளுக்கான நமது சொந்த தயார்நிலைத் திட்டம். உங்களுக்காக 150க்கும் மேற்பட்ட முன்வரையறுக்கப்பட்ட படிநிலை விளக்கங்களை உருவாக்கியுள்ளோம், எனவே உங்கள் திட்டத்தில் அம்சக் கோப்புகளை உடனடியாக எழுதத் தொடங்கலாம்.

- கட்டமைப்பு:
    - Cucumber
    - WebdriverIO
- அம்சங்கள்:
    - உங்களுக்குத் தேவையான கிட்டத்தட்ட அனைத்தையும் உள்ளடக்கிய 150க்கும் மேற்பட்ட முன்வரையறுக்கப்பட்ட படிகள்
    - WebdriverIO இன் Multiremote செயல்பாட்டை ஒருங்கிணைக்கிறது
    - சொந்த டெமோ ஆப்

## [webdriverio/jasmine-boilerplate](https://github.com/webdriverio/jasmine-boilerplate)
Babel அம்சங்களையும் பக்க பொருள்கள் முறையையும் பயன்படுத்தி WebdriverIO சோதனைகளை Jasmine உடன் இயக்குவதற்கான தயார்நிலைத் திட்டம்.

- கட்டமைப்புகள்
    - WebdriverIO
    - Jasmine
- அம்சங்கள்
    - பக்க பொருள் முறை
    - Sauce Labs ஒருங்கிணைப்பு

## [webdriverio/electron-boilerplate](https://github.com/webdriverio/electron-boilerplate)
குறைந்தபட்ச Electron பயன்பாட்டில் WebdriverIO சோதனைகளை இயக்குவதற்கான தயார்நிலைத் திட்டம்.

- கட்டமைப்புகள்
    - WebdriverIO

- அம்சங்கள்
    - Electron API போலியாக்கல்
 
## [syamphaneendra/webdriverio9-boilerplate](https://github.com/syamphaneendra/webdriverio9-boilerplate)

இந்த தயார்நிலைத் திட்டம் WebdriverIO 9 மொபைல் சோதனைகளை Cucumber, TypeScript, மற்றும் Android மற்றும் iOS தளங்களுக்கான Appium உடன், பக்க பொருள் மாதிரி முறையைப் பின்பற்றுகிறது. விரிவான பதிவு செய்தல், அறிக்கையிடல், மொபைல் சைகைகள், ஆப்-டு-வெப் வழிசெலுத்தல் மற்றும் CI/CD ஒருங்கிணைப்பு ஆகியவற்றை உள்ளடக்கியது.

- கட்டமைப்புகள்:
    - WebdriverIO v9
    - Cucumber v9
    - Appium v2.5
    - TypeScript v5

- அம்சங்கள்:
    - பல-தள ஆதரவு
      - Android (UiAutomator2)
      - iOS (XCUITest)
    - மொபைல் சைகைகள்
      - ஸ்க்ரோல்
      - ஸ்வைப்
      - நீண்ட அழுத்தம்
      - கீபோர்டை மறைத்தல்
    - ஆப்-டு-வெப் வழிசெலுத்தல்
      - சூழல் மாற்றம்
      - WebView ஆதரவு
      - உலாவி தானியங்கி (Chrome/Safari)
    - புதிய ஆப் நிலை
      - சூழ்நிலைகளுக்கு இடையே தானியங்கி ஆப் மீட்டமைப்பு
      - உள்ளமைக்கக்கூடிய மீட்டமைப்பு நடத்தை (noReset, fullReset)
    - சாதன உள்ளமைவு
      - மையப்படுத்தப்பட்ட சாதன மேலாண்மை
      - எளிதான தள மாற்றம்
    - JavaScript / TypeScript க்கான அடைவு கட்டமைப்பின் எடுத்துக்காட்டு. கீழே JS பதிப்புக்கானது, TS பதிப்பும் அதே கட்டமைப்பைக் கொண்டுள்ளது.

## [amiya-pattnaik/wdio-testgen-from-gherkin-ts](https://github.com/amiya-pattnaik/wdio-testgen-from-gherkin-ts)
Gherkin .feature கோப்புகளிலிருந்து WebdriverIO பக்க பொருள் வகுப்புகளையும் Mocha சோதனை specகளையும் தானாகவே உருவாக்குகிறது — கைமுறை முயற்சியைக் குறைக்கிறது, ஒத்திசைவை மேம்படுத்துகிறது, மற்றும் QA தானியங்கியை விரைவுபடுத்துகிறது. இந்த திட்டம் webdriver.io உடன் இணக்கமான குறியீடுகளை உருவாக்குவதோடு மட்டுமல்லாமல், webdriver.io இன் அனைத்து செயல்பாடுகளையும் மேம்படுத்துகிறது. JavaScript பயனர்களுக்கும் TypeScript பயனர்களுக்கும் இரண்டு வகைகளை உருவாக்கியுள்ளோம். ஆனால் இரண்டு திட்டங்களும் ஒரே மாதிரியாக செயல்படுகின்றன.

***இது எப்படி செயல்படுகிறது?***
- இந்த செயல்முறை இரண்டு-படி தானியங்கியை பின்பற்றுகிறது:
- படி 1: Gherkin இலிருந்து stepMap (stepMap.json கோப்புகளை உருவாக்குகிறது)
  - stepMap.json கோப்புகளை உருவாக்குகிறது:
    - Gherkin தொடரியலில் எழுதப்பட்ட .feature கோப்புகளை பகுப்பாய்வு செய்கிறது.
    - சூழ்நிலைகள் மற்றும் படிகளை பிரித்தெடுக்கிறது.
    - பின்வருவனவற்றை உள்ளடக்கிய கட்டமைக்கப்பட்ட .stepMap.json கோப்பை உருவாக்குகிறது:
      - செயல்படுத்த வேண்டிய செயல் (எ.கா., click, setText, assertVisible)
      - தர்க்க ரீதியான மேப்பிங்குக்கான selectorName
      - DOM உறுப்புக்கான தேர்வாளர்
      - மதிப்புகள் அல்லது உறுதிப்படுத்தலுக்கான குறிப்பு
- படி 2: stepMap இலிருந்து குறியீடு (WebdriverIO குறியீட்டை உருவாக்குகிறது).
  stepMap.json பின்வருவனவற்றை உருவாக்கப் பயன்படுத்துகிறது:
  - பகிரப்பட்ட முறைகள் மற்றும் browser.url() அமைப்புடன் கூடிய அடிப்படை page.js வகுப்பை உருவாக்குகிறது.
  - test/pageobjects/ உள்ளே அம்சத்திற்கு ஏற்ப WebdriverIO-இணக்கமான பக்க பொருள் மாதிரி (POM) வகுப்புகளை உருவாக்குகிறது.
  - Mocha-அடிப்படையிலான சோதனை spec-களை உருவாக்குகிறது.
- JavaScript / TypeScript க்கான அடைவு கட்டமைப்பின் எடுத்துக்காட்டு. கீழே JS பதிப்புக்கானது, TS பதிப்பும் அதே கட்டமைப்பைக் கொண்டுள்ளது.
```
project-root/
├── features/                   # Gherkin .feature files (user input / source file)
├── stepMaps/                   # Auto-generated .stepMap.json files
├── test/                 
│   ├── pageobjects/            # Auto-generated WebdriverIO tests Page Object Model classes
│   └── specs/                  # Auto-generated Mocha test specs
├── src/
│   ├── cli.js                  # Main CLI logic
│   ├── generateStepsMap.js     # Feature-to-stepMap generator
│   ├── generateTestsFromMap.js # stepMap-to-page/spec generator
│   ├── utils.js                # Helper methods
│   └── config.js               # Paths, fallback selectors, aliases
│   └── __tests__/              # Unit tests (Vitest)
├── testgen.js                  # CLI entry point
│── wdio.config.js              # WebdriverIO configuration
├── package.json                # Scripts and dependencies
├── selector-aliases.json       # Optional user-defined selector overrides the primary selector
```

    - https://search.yahoo.com/ மற்றும் http://the-internet.herokuapp.com டெமோ ஆப்ஸுடன் எடுத்துக்காட்டுகள்.
